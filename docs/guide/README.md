# 介绍

`di18n` 自动转换、基于配置的前端国际化方案，旨在提高前端国际化效率。

## 痛点 

- 重复性的工作: 提取文案 -> 起名 -> 更新国际化资源 -> 替换 
- 国际化资源和代码强耦合: 不便于非开发人员参与国际化资源维护工作
- 国际化资源管理困难: 容易出现国际化资源冗余

## 目标

- 尽可能像开发普通业务一样开发有国际化需求的业务
- 国际化资源和源代码解耦

> 本方案主要解决文案抽取和更新问题，因国际化引入的布局问题仍需我们自己解决。

## 特性

- 🤖 自动提取中文字符，并替换为资源 key
- 📦 资源文件和代码解耦
- 🚀 发布即更新，无需上线
